序列化 反序列化

web学习笔记 序列化 反序列化

题目为unserialize，

class xctf{
public $flag = '111';
public function __wakeup(){
exit('bad requests');
}
?code=

题目暗示，试将对象xctf 序列化：

<?php
class xctf {
    public $flag = '111';
    public funciton_wakeup() {
        exit('bad request');
    }
}
$c = new xctf();
echo(serialize($c));
?>

得到序列化字符串：

O:4:”xctf”:1:{s:4:”flag”;s:3:”111”;}

含义为：

Object : length: “class name” : n : { String : name1 : value1 ; String : name2 : value2; ……}

length: 被序列化对象的名称长度；
n：被序列化对象的属性个数；

绕过

为了不要’bad request’，需绕过_wakeup() 函数
如何绕过：当序列化字符串中的属性个数值大于实际的属性个数时，就会导致反序列化异常，从而跳过_wakeup()
针对此题修改 n > 1 即可
用 POST 提交，得flag

1、serialize()函数：用于序列化对象或数组，并返回一个字符串。序列化对象后，可以很方便的将它传递给其他需要它的地方，且其类型和结构不会改变。

2、unserialize()函数：用于将通过serialize()函数序列化后的对象或数组进行反序列化，并返回原始的对象结构。

总之，序列化和反序列化，是对象的保存和重建。

3、魔术方法：PHP 将所有以 __（两个下划线）开头的类方法保留为魔术方法。所以在定义类方法时，除了上述魔术方法，建议不要以 __ 为前缀。

4、serialize()和unserialize()函数对魔术方法的处理：serialize()函数会检查类中是否存在一个魔术方法__sleep()。 如果存在，该方法会先被调用，然后才执行序列化操作。

5、__wakeup()执行漏洞：一个字符串或对象被序列化后，如果其属性被修改，则不会执行__wakeup()函数，这也是一个绕过点。

BTW 三个不同类型的字符串，但值相同，
序列化后输出的值也不相同 

<?php
class test{
    private $test1="hello";
    public $test2="hello";
    protected $test3="hello";
}
$test = new test();
echo serialize($test); 
?>

O:4:”test”:3:{s:11:”test test1”;s:5:”hello”;s:5:”test2”;s:5:”hello”;s:8:” * test3”;s:5:”hello”;}
