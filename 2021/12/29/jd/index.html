京东品牌情感分析

京东商品情感分析

基于BosonNLP
情感词字典，爬取某类商品信息，包括商品价格、商品品牌、商品，对比不同价位的品牌性价比。

python 3.7 + BeautifulSoup + jieba + BosonNLP + Selenium

https://github.com/ppphhhleo/JD_brandSenti

QUICK START

run.py

  • 获取京东商品基本信息，可输入搜索的关键词，如“电脑”、“单反相机”

  • 获取商品的众多品牌

  • 获取每个商品的评论信息

  • 对评论分析情感极性

  • 综合商品的评论情感极性，计算商品的情感值

  • 对爬取的所有商品，进行价格排序

  • 设置价位，将同一价位的品牌合并

  • 同价位内部品牌，进行情感值排序



方案设计

总体上分为三个步骤，即数据的爬取和处理，模型和方法的选择，以及利用模型进行情感分析并对结果进行处理。整体流程图如下：

选取**“笔记本电脑”**作为爬取的主要商品品类。

  • 基于情感词典BosonNLP

对于数据的爬取和处理部分，首先我们需要获取大量商品信息，在京东搜索结果页面中包含较多商品单元，对每个单元，首先爬取其商品名、商品价格、商品评价数、店铺名、详细页面链接。将初步获取的单元信息，存储至data.csv文件中；其次，由于京东商品的品牌名和具体评论在详细页面链接（二级链接），需要依次对以获取的详细页面链接进行访问，爬取商品对应的品牌名，将商品名、价格、品牌名，存入brand.csv文件中。以同样方式，在详细页面中爬取商品对应的100条评论数据，将详细链接、评论数据存入comment.csv中。

接下来，是对评论的情感分析部分，对已经获取到的商品价格、商品品牌和商品评论信息，我们需要确定对短文本评论进行情感分析的策略和方法。当前，有基于情感词的方法和基于模型分类的方法。本次实验将选取基于情感词计算的方法进行分析。对于以获取的商品评论，我们逐条进行情感值计算，将评论和对应的情感值、对应的情感倾向，存入comment_emotion.csv中。此前单个商品爬取评论数为50条，因此，需要每50条计算平均值，作为该商品的综合情感值，将商品名、价格、品牌、综合情感值存入brand_emotion.csv文件中。

最后，根据已获取的商品情感值、商品价格、商品品牌，进行数据分析，分析出在不同价位中各个品牌的商品评价好感度梯度排名，进而可得知在不同预算下购买该品类商品可供选择的较佳品牌。

具体流程如下：

  • 基于神经网络

    本次实验，将选用TextCNN作为文本情感分类模型。数据获取和处理，方式与以上方法相同，除此之外，需要定义、训练模型，定义情感值。

实现过程


0 爬取数据

根据以上流程分析，爬取数据可分成三阶段：爬取商品基本信息，爬取商品品牌，爬取商品评论。

1、商品集合网页

针对京东商品进行爬虫。首先我们要对京东的 url
结构进行解析。打开京东，搜索“笔记本电脑”关键字，然后产看其 url 结构：

部分为统一格式。Keyword部分，是我们输入的关键词，对搜索主页进行分析，搜索框输入为**#key**，则可通过webdrive输入需要搜索的商品类目。输入后，模拟点击搜索键button。

点击搜索后，页面有多个商品单元，我们分析单个商品单元的元素组成，根据所需获取的商品信息找到对应的网页元素：p-img包含href详细链接，p-price包含价格，p-name包含商品名称，p-commit包含商品评论总数，J_im_icon包含商店名。

通过find_element_by_css_selector依次获取以上商品基本信息，为保证后续爬取
商品评论数足够，我们根据商品的评价总数进行筛选，仅保存评论总数过万的商品基本信息。

浏览完页面的所有商品后，可点击页面底部bottomPage栏目中的下一页按钮pn-next，模拟翻页。

设定待爬取商品的总数，每个商品以（商品名，价格，评论总数，商店名，详细链接）为一个记录存储至data.csv，作为商品基本信息。

2、商品详细页面

由于商品对应的品牌名和评论并不展示在以上搜索结果的页面，需要在商品详细页面中获取，因此，需对爬取所有商品逐条浏览详细链接，获取对应的品牌名。

在商品详细页面中，对应品牌名存储在**class = ‘p-parameter’/ id =
‘parameter-brand’**元素中。

对于商品品牌，以（商品名称，价格，品牌名）为一条记录存储至brand.csv文件。

商品详细页面中，要获取商品评价，需要模拟点击“商品评价”按键，我们找到按键对应的元素如图所示。

点击后，可见多个商品评价的集合，单个评价元素表示为class
=“comment-item”，单个评论页面中含有10个评论左右。

展开查看单个评论内部的信息结构，class =
“comment-con”表示单条评论的所有文字，这是我们后续情感分析所需要内容。我们先获取单个页面所有评论集合comment-con，依次存储其中的文字内容text。

对于单个商品，我们获取50条评论进行分析，评论页的“下一页”按键由“ui-pager-next”表示。

为保证商品覆盖品牌较全面，又能较短时间内完成商品评论爬取，确定爬取商品数量为120个，将读取商品基本信息data.csv文件，爬取其中120个商品的评论信息，每个商品50条评论，依次存储记录在comment.csv文件，最终获取120
* 50 = 6000条商品评论。

至此，已完成对所需商品数据的获取，包含商品基本信息data.csv（商品名，价格，评论总数，商店名，详细链接），商品品牌信息brand.csv（商品名称，价格，品牌名），商品评论信息comment.csv（评论），接下来将对数据进行处理。


1 情感分析

1、数据预处理

获取评论数据后，还无法直接进行情感分析，需要逐条读取评论数据，删除停用词，并利用jieba进行分词。

停用词词典使用baidu_stopwords，包含与情感的直接表达无关的字词：


2、情感值计算

本次实验中，基于情感词分析，我采用了BosonNLP词典进行计算情感值，方法较为清晰简洁，在单条句子文本分词后，将分词好的列表词数据与BosonNLP词典进行逐个匹配，记录匹配得到的情感词分值，最后统计得到的分值总和，代表句子情感值，如分值大于0，表示该句子情感倾向为积极；如小于0，则表示情感倾向为消极。BosonNLP词典共有10万以上的情感词及其情感值。

具体流程为：

我们读取comment.csv文件，逐条读取评论，并计算句子级情感值，判断情感倾向，最终以**（评论，情感值，情感极性）为记录保存单个评论情感emotion.csv文件**。

评论的情感值计算如下所示，因评论基本为好评，以BosonNLP词典进行情感值计算，若文本越长，则情感值绝对值越大。单个评论，及其情感值，如下所示：

对于每个商品，我们分别爬取并计算了50条评论的情感值，为了全面综合地描述商品的情感反馈，可以计算50条评论的情感平均值作为该商品的情感评价得分，将**（商品名称，商品价格，商品品牌，情感值）**作为品牌在该价位的商品评价极性保存至brand_emotion.csv

至此，我们已完成对所有商品的评价信息进行情感值计算和情感极性分析，获得如下所示的品牌价格与情感值数据，并未对价格或情感值进行排序。


2 数据分析

1、价格排序

读取brand_emotion.csv文件的商品，及其对应的价格和情感值，因后续需要对商品进行价格区段的划分，因此需要对价格进行排序，借助sorted函数排列，同时将原本的商品信息对应关系保留，将（商品名，价格，品牌，情感值）作为记录保存。

2、品牌合并与对比

我们将商品按照价格升序进行排列后，可以看到位于同一价位的同一品牌，也有多种商品，并且针对电子产品，会出现同一品牌产品的不同配置，因此需要将特定价位中单一品牌所有产品的情感值合并，这里取平均值作为该品牌在该价位的商品情感值。

根据已有的商品价格升序数据，这里粗略将笔记本电脑的商品价格分为7个价位，依次为：0~3000、3000~4000，4000~5000，5000~6000，6000~7000，7000~8000，>8000，Counts数组依次统计每个价位的商品数。

创建一个列表用于存储我们已经处理对比过的品牌，如该品牌在之前已经被处理过了，那么则跳过此商品元素；如果在这个价位中，这个品牌还未被对比过，就把该品牌加入到已处理的列表中，并用这个数据（该索引下的价位和品牌）进行对比，如果品牌名相同，则将情感值累加，之后取平均值，得到该价位的品牌情感值。

最后将（价格，品牌，情感）作为记录保存，便可以按照价位、情感值进行排序分析，如所得图表所示。
